<a href="#using-the-alchemy-sdk" id="using-the-alchemy-sdk" style="color: inherit; text-decoration: none;">
  <h2>Using the Alchemy SDK</h2>
</a>
<p>The Alchemy SDK currently supports the following namespaces:</p>
<ul>
<li><code>core</code>: All commonly-used Ethers.js Provider methods and Alchemy Enhanced API methods</li>
<li><code>nft</code>: All Alchemy NFT API methods</li>
<li><code>ws</code>: All WebSockets methods</li>
<li><code>transact</code>: All Alchemy Transaction API methods</li>
<li><code>notify</code>: CRUD endpoints for modifying Alchemy Notify Webhooks</li>
<li><code>debug</code>: Methods to inspect and replay transactions and blocks</li>
</ul>
<p>If you are already using Ethers.js, you should be simply able to replace the Ethers.js Provider object with <code>alchemy.core</code> and it should work properly.</p>
<blockquote>
<p><strong>‚ÑπÔ∏è ENS Name Resolution</strong></p>
<p>The Alchemy SDK now supports ENS names (e.g. <code>vitalik.eth</code>) for every parameter where you can pass in a Externally Owned Address, or user address (e.g. <code>0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045</code>).</p>

<p>The Alchemy SDK also supports a number of Ethers.js objects that streamline the development process:</p>
<ul>

</ul>

<a href="#alchemy-settings" id="alchemy-settings" style="color: inherit; text-decoration: none;">
  <h2>Alchemy Settings</h2>
</a>
<p>An <code>AlchemySettings</code> object can be passed on instantiation to the Alchemy object, with the following optional parameters:</p>
<ul>
<li><code>apiKey</code>: API key that can be found in the Alchemy dashboard. Defaults to <code>demo</code>: a rate-limited public key.</li>
<li><code>network</code>: Name of the network. Defaults to <code>Network.ETH_MAINNET</code></li>
<li><code>maxRetries</code>: The maximum number of retries to attempt if a request fails. Defaults to 5.</li>
<li><code>url</code>: Optional URL endpoint to use for all requests. Setting this field will override the URL generated by the <code>network</code> and<code>apiKey</code> fields.</li>
<li><code>authToken</code>: Alchemy auth token required to use the Notify API. This token can be found in the Alchemy Dashboard on the Notify tab.</li>
<li><code>batchRequests</code>: Optional setting that automatically batches and sends json-rpc requests for higher throughput and reduced network IO. Defaults to false.</li>
<li><code>requestTimeout</code>: Optional setting that sets the timeout for requests in milliseconds for the NFT and Notify namespaces. Defaults to no timeout.</li>
</ul>

<a href="#alchemy-core" id="alchemy-core" style="color: inherit; text-decoration: none;">
  <h2>Alchemy Core</h2>
</a>
<p>The core namespace contains all commonly-used <a href="https://docs.ethers.io/v5/api/providers/api-providers/#AlchemyProvider">Ethers.js Provider</a> methods. If you are already using Ethers.js, you should be simply able to replace the Ethers.js Provider object with <code>alchemy.core</code> when accessing provider methods and it should just work.</p>
<p>It also includes the majority of Alchemy Enhanced APIs, including:</p>
<ul>
<li><code>getTokenMetadata()</code>: Get the metadata for a token contract address.</li>
<li><code>getTokenBalances()</code>: Gets the token balances for an owner given a list of contracts.</li>
<li><code>getAssetTransfers()</code>: Get transactions for specific addresses.</li>
<li><code>getTransactionReceipts()</code>: Gets all transaction receipts for a given block.</li>
</ul>
<p>You will also find the following utility methods:</p>
<ul>
<li><code>findContractDeployer()</code>: Find the contract deployer and block number for a given contract address.</li>
<li><code>getTokensForOwner()</code>: Get all token balances and metadata for a given owner address</li>
</ul>

class="hl-1">.</span><span class="hl-0">ws</span><span class="hl-1">.</span><span class="hl-7">removeAllListeners</span><span class="hl-1">();</span>
</code></pre>
<p>The SDK brings multiple improvements to ensure correct WebSocket behavior in cases of temporary network failure or
dropped connections. As with any network connection, you should not assume that a WebSocket will remain open forever
without interruption, but correctly handling dropped connections and reconnection by hand can be challenging to get
right. The Alchemy SDK automatically handles these failures with no configuration necessary. The main benefits are:</p>
<ul>
<li>Resilient event delivery: Unlike standard Web3.js or Ethers.js, you will not permanently miss events which arrive
while the backing WebSocket is temporarily down. Instead, you will receive these events as soon as the connection
is reopened. Note that if the connection is down for more than 120 blocks (approximately 20 minutes), you may
still miss some events that were not part of the most recent 120 blocks.</li>
<li>Lowered rate of failure: Compared to standard Web3.js or Ethers.js, there are fewer failures when sending requests
over the WebSocket while the connection is down. The Alchemy SDK will attempt to send the requests once the connection
is reopened. Note that it is still possible, with a lower likelihood, for outgoing requests to be lost,
so you should still have error handling as with any network request.</li>
</ul>

<a href="#alchemy-transact" id="alchemy-transact" style="color: inherit; text-decoration: none;">
  <h2>Alchemy Transact</h2>
</a>
<p>The <code>transact</code> namespace contains methods used for simulating and sending transactions. The unique methods to the <code>transact</code> namespace are:</p>
<ul>
<li><code>sendPrivateTransaction()</code>: Send a private transaction through Flashbots.</li>
<li><code>cancelPrivateTransaction()</code>: Cancel a private transaction sent with Flashbots.</li>
<li><code>simulateAssetChanges()</code>: Simulate a transaction and get a list of asset changes.</li>
<li><code>simulateExecution()</code>: Simulate a transaction and get a full list of internal transactions, logs, ABI decoded results and more.</li>
<li><code>simulateAssetChangesBundle()</code>: Simulate a list of transactions in sequence and get a list of asset changes.</li>
<li><code>simulateExecutionBundle()</code>: Simulate a list of transactions in sequence and get a full list of internal transactions, logs, ABI decoded results and more.</li>
</ul>
<p>The <code>transact</code> namespace also aliases over several commonly used methods from the <code>core</code> namespace for convenience:</p>
<ul>
<li><code>getTransaction()</code>: Returns the transaction for the given transaction hash.</li>
<li><code>sendTransaction()</code>: Sends a standard transaction to the network to be mined.</li>
<li><code>waitForTransaction()</code>: Waits for a transaction to be mined and returns the transaction receipt.</li>
</ul>

<a href="#alchemy-nft-api" id="alchemy-nft-api" style="color: inherit; text-decoration: none;">
  <h2>Alchemy NFT API</h2>
</a>
<p>The SDK currently supports the following <a href="https://docs.alchemy.com/alchemy/enhanced-apis/nft-api">NFT API</a> endpoints
under the <code>alchemy.nft</code> namespace:</p>
<ul>
<li><code>getNftMetadata()</code>: Get the NFT metadata for an NFT contract address and tokenId.</li>
<li><code>getNftMetadataBatch()</code>: Get the NFT metadata for multiple NFT contract addresses/token id pairs.</li>
<li><code>getContractMetadata()</code>: Get the metadata associated with an NFT contract</li>
<li><code>getContractMetadataBatch()</code>: Get the metadata associated with multiple NFT contracts in a single request.</li>
<li><code>getContractsForOwner()</code>: Get all NFT contracts that the provided owner address owns.</li>
<li><code>getNftsForOwner()</code>: Get NFTs for an owner address.</li>
<li><code>getNftsForOwnerIterator()</code>: Get NFTs for an owner address as an async iterator (handles paging automatically).</li>
<li><code>getNftsForContract()</code>: Get all NFTs for a contract address.</li>
<li><code>getNftsForContractIterator()</code>: Get all NFTs for a contract address as an async iterator (handles paging
automatically).</li>
<li><code>getOwnersForNft()</code>: Get all the owners for a given NFT contract address and a particular token ID.</li>
<li><code>getOwnersForContract()</code>: Get all the owners for a given NFT contract address.</li>
<li><code>getMintedNfts()</code>: Get all the NFTs minted by the owner address.</li>
<li><code>getTransfersForOwner()</code>: Get all the NFT transfers for a given owner address.</li>
<li><code>getTransfersForContract()</code>: Get all the NFT transfers for a given NFT contract address.</li>
<li><code>verifyNftOwnership()</code>: Check whether the provided owner address owns the provided NFT contract addresses.</li>
<li><code>isSpamContract()</code>: Check whether the given NFT contract address is a spam contract as defined by Alchemy (see the <a href="https://docs.alchemy.com/alchemy/enhanced-apis/nft-api/nft-api-faq#nft-spam-classification">NFT API FAQ</a>)</li>
<li><code>getSpamContracts()</code>: Returns a list of all spam contracts marked by Alchemy.</li>
<li><code>reportSpam()</code>: Report feedback that a given NFT contract address is a spam contract as defined by Alchemy.</li>
<li><code>isAirdropNft()</code>: Check whether the given NFT token is marked as an airdrop or not. Airdrops are defined as NFTs that were minted to a user address in a transaction sent by a different address.</li>
<li><code>refreshNftMetadata()</code>: Refresh the cached NFT metadata for a contract address and a single tokenId.</li>
<li><code>refreshContract()</code>: Enqueues the specified contract address to have all token ids&#39; metadata refreshed.</li>
<li><code>getFloorPrice()</code>: Return the floor prices of a NFT contract by marketplace.</li>
<li><code>computeRarity()</code>: Get the rarity of each attribute of an NFT.</li>
<li><code>getNftSales()</code>: Returns NFT sales that have happened through on-chain marketplaces.</li>
<li><code>summarizeNftAttributes()</code>: Get the summary of attribute prevalence for all NFTs in a contract.</li>
<li><code>searchContractMetadata()</code>: Search for a keyword across metadata of all ERC-721 and ERC-1155 smart contracts.</li>
</ul>

<a href="#pagination" id="pagination" style="color: inherit; text-decoration: none;">
  <h3>Pagination</h3>

the

and <code>getNftsForContractIterator()</code> functions that automatically paginate through all NFTs and yields them via

<a href="#sdk-vs-api-differences" id="sdk-vs-api-differences" style="color: inherit; text-decoration: none;">
  <h3>SDK vs API Differences</h3>
</a>
<p>The NFT API in the SDK standardizes response types to reduce developer friction, but note this results in some
differences compared to the Alchemy REST endpoints:</p>
<ul>
<li>Methods referencing <code>Collection</code> have been renamed to use the name <code>Contract</code> for greater accuracy: e.g. <code>getNftsForContract</code>.</li>
<li>Some methods have different naming that the REST API counterparts in order to provide a consistent API interface (
e.g. <code>getNftsForOwner()</code> is <code>alchemy_getNfts</code>, <code>getOwnersForNft()</code> is <code>alchemy_getOwnersForToken</code>).</li>
<li>SDK standardizes to <code>omitMetadata</code> parameter (vs. <code>withMetadata</code>).</li>
<li>Standardization to <code>pageKey</code> parameter for pagination (vs. <code>nextToken</code>/<code>startToken</code>)</li>
<li>Empty <code>TokenUri</code> fields are omitted.</li>
<li>Token ID is always normalized to an integer string on <code>BaseNft</code> and <code>Nft</code>.</li>
<li>Some fields omitted in the REST response are included in the SDK response in order to return an <code>Nft</code> object.</li>
<li>Some fields in the SDK&#39;s <code>Nft</code> object are named differently than the REST response.</li>
</ul>

<a href="#alchemy-notify" id="alchemy-notify" style="color: inherit; text-decoration: none;">
  <h2>Alchemy Notify</h2>
</a>
<p>The <a href="https://docs.alchemy.com/reference/notify-api-quickstart">Alchemy Notify API</a> helps developers set up webhooks in their apps. The namespace provides methods to programmatically create, read, update, and delete your webhooks along with typings for the different webhooks. To learn more about Webhooks, please refer to the <a href="https://docs.alchemy.com/reference/notify-api-quickstart#what-are-webhooks">Alchemy documentation</a>.</p>
<p>Methods on the <code>NotifyNamespace</code> can be accessed via <code>alchemy.notify</code>. To use the methods, you must include your team&#39;s auth token in the <code>authToken</code> field of <code>AlchemySettings</code> when instantiating the SDK. The auth token can be found on the Alchemy Dashboard in the Notify Tab.</p>
<p>Methods include:</p>
<ul>
<li><code>getAllWebhooks()</code>: Get all webhooks on your team.</li>
<li><code>getAddresses()</code>: Get all addresses tracked for the provided Address Activity Webhook.</li>
<li><code>getNftFilters()</code>: Get all NFT filters tracked for the provided NFT Activity Webhook.</li>
<li><code>createWebhook()</code>: Create a new webhook.</li>
<li><code>updateWebhook()</code>: Update an existing webhook&#39;s active status or tracked addresses and NFT filters.</li>
<li><code>deleteWebhook()</code>: Delete the provided webhook.</li>
</ul>

<a href="#alchemy-debug" id="alchemy-debug" style="color: inherit; text-decoration: none;">
  <h2>Alchemy Debug</h2>
</a>
<p>Methods on the <code>DebugNamespace</code> can be accessed via <code>alchemy.debug</code>. These methods are used for inspecting and debugging transactions.</p>
<p>Methods include:</p>
<ul>
<li><code>traceCall()</code>: Run an <code>eth_call</code> with the context of the provided block execution using the final state of the parent block as the base.</li>
<li><code>traceTransaction()</code>: Run the transaction in the exact same manner as it was executed on the network. It will replay any transaction that may have been executed prior to this one before it and will then attempt to execute the transaction that corresponds to the given hash.</li>
<li><code>traceBlock()</code>: Replay a block that has already been mined.</li>
</ul>
